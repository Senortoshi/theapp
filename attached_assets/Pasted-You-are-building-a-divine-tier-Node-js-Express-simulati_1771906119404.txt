═══════════════════════════════════════
API ENDPOINTS
═══════════════════════════════════════

POST /contribute
  body: { username, content, type? }
  - Score the contribution
  - Run decay on all existing contributions
  - Check for references/boosts
  - Recalculate all shares
  - Emit WS event: 'contribution_added', 'shares_updated'
  - Return: full contributor state + their new share

GET /shares
  - Returns all contributors sorted by share desc
  - Include: share_pct, satoshis_earned, status, share_history (last 20 points)
  - Include: global stats { total_contributors, total_revenue, total_contributions }

POST /revenue  
  body: { amount_satoshis }
  - Distribute revenue
  - Return full distribution breakdown

GET /payouts
  - Full chronological payout history
  - Per contributor breakdown
  - Running totals

GET /leaderboard
  - Top contributors by: share%, satoshis earned, contributions count, avg score
  - Include a "momentum" field: rate of share change last 60 seconds

GET /simulate/run
  - Triggers the auto-simulation (see below)
  - Returns simulation started confirmation

GET /state
  - Full snapshot of everything for initial page load

On server start, seed with this narrative arc:
  
  Phase 1 — Genesis (5 contributors, 8 contributions)
  A small team starts building. Shares are relatively even.
  
  Phase 2 — Acceleration (10 more contributors over 30 seconds)
  New contributors enter. Some build on existing ideas (triggering boosts).
  Original contributors start to see share shifts.
  Early contributors with referenced ideas hold strong.
  
  Phase 3 — Scale (20 more contributors, revenue flowing)
  Revenue stream begins at 500 satoshis every 3 seconds.
  A star contributor emerges with synthesis-type contributions.
  Some early contributors decay. New builders rise.
  The fairness engine visibly breathes — nothing is static.

Auto-simulation loop (when /simulate/run is called):
  - Every 4 seconds: add a new simulated contribution
  - Every 3 seconds: inject 500 satoshis revenue
  - Every 8 seconds: one contribution "builds on" a random previous one
  - Run for 3 minutes then loop
  - Emit all events via WebSocket in real time
